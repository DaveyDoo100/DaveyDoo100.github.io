Unit 1 & 2 Vocabulary

Unit 1: Data

  • Bit: A binary digit, either 0 or 1.
  • Byte: A sequence of 8 bits.
  • Roundoff: Error when not enough bits represent a number with precision.
  • Analog Data: Values that change smoothly, like music or light.
  • Lossless: Compression that preserves all data exactly.
  • Lossy: Compression that discards some data permanently.
  • Metadata: Data about data, such as file details.

Unit 2: Computer Systems and Networks

  • Computing Device: Any device that runs a program, like a phone or computer.
  • Computer Network: A group of connected devices exchanging data.
  • Path: The series of connections between devices in a network.
  • Protocol: Rules that define how systems communicate.
  • Fault Tolerant: Keeps working even if some parts fail.
  • Scalability: Ability of a system to grow or shrink as needed.
  • Bandwidth: Maximum data transfer rate, measured in bits per second.
  • URL: A readable web address, like www.example.com.

Internet Structure and Layers

  • Physical Network: The hardware carrying internet signals.
  • Internet Protocol (IP): Assigns addresses and routes data.
  • TCP: Reliable, error-checked delivery of data packets.
  • UDP: Faster but less reliable alternative to TCP.
  • Packets: Small chunks of data messages are split into.
  • Routers: Devices that forward data using the best path.
  • DNS: Translates domain names into IP addresses.
  • HTTP: Protocol for web pages and media.
  • WWW: Linked system of web pages and media browsable online.

Key Concepts

  • Message Delivery: Messages split into packets that may take different paths.
  • Redundancy: Multiple pathways ensure reliability.
  • IPv4: Uses 32-bit addresses (~4.3 billion unique addresses).
  • IPv6: Adds 16 more bits, expanding address space hugely.

How the Internet Works

The internet is a network of networks that connects billions of devices across the world. Messages are broken into smaller packets, sent through multiple paths, and reassembled when they reach the destination. This system makes the internet fault tolerant, scalable, and powerful enough to handle enormous amounts of data.
